## 核心概念
Superpowers 是一个为 AI Agent 提供实体状态管理与上下文增强的工具集。它通过从对话和非结构化文本中自动提取实体，并将其持久化为结构化知识，解决 LLM 长期记忆和上下文窗口受限的问题。其核心在于将无序的对话历史转化为可查询的实体图谱，使 Agent 能够具备跨会话的状态感知能力。

## 设计原理
设计原理基于将 LLM 的无状态推理转化为有状态的实体交互。通过在 Agent 的执行循环中拦截和解析输入输出，利用 LLM 提取关键实体及其属性变化，并将这些变化以补丁的形式应用到本地的 JSON 文档或知识库中。在后续交互中，系统将相关实体的最新状态作为隐式上下文注入 Prompt，从而避免传入冗长的完整对话历史，降低 Token 消耗并提高指令遵循度。

## 关键实现
关键实现依赖于实体提取 Prompt 模板和本地状态存储。Agent 每次交互后，系统调用底层 LLM 解析出新增或变更的实体（如人物、项目、任务），将其序列化为结构化 JSON 并通过文件系统或数据库持久化。检索时采用基于关键词或向量的匹配算法，将 Top-K 相关实体的当前状态注入到系统提示词中。接口层提供标准化的工具调用封装，支持无缝集成到主流 Agent 框架（如 LangChain 或自定义 ReAct 循环）中。

## 关联分析
与 Mem0（专注于用户记忆管理）、Zep（长期记忆图引擎）和 LangChain Memory 模块属于同类项目。相比于 Zep 强调图数据库的深度集成，Superpowers 更轻量，倾向于使用本地文件系统进行快速原型验证；相比于 Mem0 侧重于偏好记忆，它更强调对复杂实体属性和关系的动态追踪。

## 可执行建议
1. 克隆仓库并在本地运行示例脚本，观察 Agent 在多轮对话中如何自动提取并更新实体 JSON 文件。2. 将其核心的实体提取 Prompt 提取出来，接入你现有的 Agent 工作流，对比引入实体状态注入前后的 Token 消耗与上下文一致性表现。
